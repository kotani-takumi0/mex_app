# MEX App 使用ガイド

過去のプロジェクト知識を活用して、新しい開発アイデアの質を上げるAI壁打ちアプリです。
過去の没案（不採用プロジェクト）から学び、同じ失敗を繰り返さないためのナレッジ基盤を構築します。

---

## 前提条件

アプリを使い始める前に、以下のセットアップが完了している必要があります。
セットアップ手順は [README.md](./README.md) を参照してください。

- バックエンド（FastAPI）が `http://localhost:8000` で起動中
- フロントエンド（React）が `http://localhost:3000` で起動中
- PostgreSQL・Qdrant が稼働中
- OpenAI API キーが設定済み

---

## アカウント登録・ログイン

### 新規登録

1. ブラウザで `http://localhost:3000` を開く
2. ランディングページの「無料で始める」ボタンをクリック
3. 認証ページで「登録」タブを選択
4. 以下を入力して登録
   - **表示名**: ダッシュボードに表示される名前
   - **メールアドレス**: ログインに使用
   - **パスワード**: 任意のパスワード
5. 登録完了後、自動でダッシュボードに遷移

### ログイン

1. 認証ページで「ログイン」タブを選択
2. 登録済みのメールアドレスとパスワードを入力
3. ログインボタンをクリック

> ログイン状態はブラウザに保持されます。明示的にログアウトするまで再ログインは不要です。

## ダッシュボード（`/dashboard`）

ログイン後に最初に表示される画面です。アプリ全体の利用状況を一覧できます。

### 表示内容

| 項目 | 説明 |
|------|------|
| **壁打ち回数（本日）** | 今日実行したアイデア壁打ちの回数 |
| **振り返り回数（今月）** | 今月登録した振り返りの件数 |
| **ナレッジ総数** | 蓄積された過去プロジェクトの総件数 |

### 最近のプロジェクト

直近に登録・壁打ちしたプロジェクトがカード形式で表示されます。
ここから各機能へのショートカットも利用できます。

## アイデア壁打ち（`/sparring`）

MEX App の中核機能です。新しいアイデアをAIに壁打ちし、過去の知見に基づいたフィードバックを受け取ります。

### 使い方

#### ステップ1: アイデアを入力

ナビゲーションバーの「壁打ち」をクリックし、以下のフォームに情報を入力します。

| 入力項目 | 内容 | 必須 |
|---------|------|------|
| **目的・概要** | アイデアの目的や解決したい課題 | はい |
| **ターゲット** | 想定するユーザー・市場 | はい |
| **ビジネスモデル** | マネタイズ方法・収益構造 | はい |
| **補足情報** | その他の背景や制約条件 | いいえ |

入力が完了したら「壁打ちを開始」ボタンをクリックします。

#### ステップ2: AI分析結果を確認

AIがバックグラウンドで以下の処理を実行し、結果を表示します。

1. **類似ケース**: 入力したアイデアに類似する過去のプロジェクトが一覧表示されます
   - 各ケースには「採用」「不採用」のラベルと類似度スコアが付きます
   - 過去にどんなプロジェクトが成功・失敗したかを比較できます

2. **懸念ポイント**: 過去の失敗パターンから抽出された注意点です
   - カテゴリ別（財務、運用、市場など）にグルーピングされます
   - 出現頻度が高いほど重要度が高い傾向があります

3. **クリティカルな問い**: AIが生成した、アイデアを深掘りするための質問です
   - 各質問には「なぜこの質問が重要か」の根拠が添えられます
   - 関連する失敗パターンも表示されます

#### ステップ3: 問いに回答する

クリティカルな問いに対して、画面上で直接回答を入力できます。

- 回答は自動保存され、進捗バーに反映されます
- 全ての問いに回答する必要はありませんが、回答が多いほどアイデアの解像度が上がります
- 回答した内容は後から見返すことができます

### 活用のコツ

- **できるだけ具体的に書く**: 「SNSアプリ」ではなく「30代の共働き世帯向け家事分担SNS」のように、ターゲットと課題を明確に
- **ビジネスモデルを言語化する**: 収益の流れを具体的に書くことで、より的確な懸念点が提示されます
- **問いを活用する**: 答えに窮する質問ほど、検討が不十分な領域を示しています

## 振り返り（`/retrospective`）

プロジェクトの結果や学びを記録し、将来の壁打ちで活用されるナレッジとして蓄積します。

### 使い方

ナビゲーションバーの「振り返り」をクリックし、フォームに以下を入力します。

#### 基本情報

| 入力項目 | 内容 |
|---------|------|
| **プロジェクト名** | プロジェクトの識別名 |
| **結果** | completed（完了）/ shipped（リリース）/ withdrawn（撤退）/ cancelled（中止）/ pending（保留）から選択 |

#### 意思決定の記録

| 入力項目 | 内容 |
|---------|------|
| **判断** | continue（継続）/ pivot（方針転換）/ stop（中止）から選択 |
| **判断理由** | なぜその判断に至ったかの詳細 |

#### 学びの記録

複数の学びを登録できます。

| 入力項目 | 内容 |
|---------|------|
| **トピック** | 学びの対象領域（例：技術選定、チーム体制、市場調査） |
| **インサイト** | 具体的な気づきや教訓 |

#### 議論サマリー

プロジェクトの振り返り会議やレビューでの議論内容をテキストで記録します。

### 登録後の動作

「振り返りを登録」ボタンをクリックすると、以下が実行されます。

1. 入力内容からベクトル埋め込みが生成される
2. 失敗パターンが自動タグ付けされる（該当する場合）
3. PostgreSQL にプロジェクトデータが保存される
4. Qdrant にベクトルが保存され、将来の類似検索に活用可能になる

成功画面では、割り当てられた失敗パターンが確認できます。

### 蓄積のコツ

- **成功プロジェクトも登録する**: 不採用・中止案件だけでなく、成功事例も登録することで壁打ちの精度が上がります
- **判断理由を丁寧に書く**: 「うまくいかなかった」ではなく「ターゲット市場の規模が想定の1/10だった」のように具体的に
- **こまめに登録する**: プロジェクト終了直後が最も記憶が鮮明です。後回しにせず、終了時点で振り返りを記録しましょう

## 画面遷移フロー

```
ランディングページ (/)
  │
  ├─ [無料で始める] → 認証ページ (/auth)
  │                      ├─ 登録 → ダッシュボード (/dashboard)
  │                      └─ ログイン → ダッシュボード (/dashboard)
  │
  └─ ※認証済みの場合 → ダッシュボード (/dashboard)
                          │
                          ├─ [壁打ちを始める] → アイデア壁打ち (/sparring)
                          │                       └─ 結果表示（同画面内）
                          │
                          └─ [振り返りを記録] → 振り返り (/retrospective)
                                                  └─ 登録完了画面（同画面内）
```

## よくある質問

### Q: ナレッジがゼロの状態で壁打ちしても意味がありますか？

初回は類似ケースが見つからないため、AIが一般的な観点から問いを生成します。まずは振り返り機能で過去のプロジェクトを数件登録してから壁打ちを試すと、より有益な結果が得られます。

### Q: 他のユーザーのナレッジも参照されますか？

現在はユーザーごとにナレッジが分離されています。自分が登録した振り返りのみが壁打ちの検索対象になります。

### Q: 壁打ち結果は保存されますか？

はい。壁打ちの結果と問いへの回答は自動保存されます。ダッシュボードの「最近のプロジェクト」から過去の壁打ち結果を確認できます。

### Q: Free プランと Pro プランの違いは？

現在、プランによる機能制限は開発中です。Free プランでも全機能を利用できます。
