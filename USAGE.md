# MEX App 使用ガイド

MEX App は AI 支援開発の「過程」と「理解度」を可視化するポートフォリオプラットフォームです。
プロジェクトごとの開発ログを蓄積し、公開ポートフォリオとして共有できます。

---

## 前提条件

アプリを使い始める前に、以下のセットアップが完了している必要があります。
セットアップ手順は [README.md](./README.md) を参照してください。

- バックエンド（FastAPI）が `http://localhost:8000` で起動中
- フロントエンド（React）が `http://localhost:3000` で起動中
- PostgreSQL・Qdrant が稼働中
- OpenAI API キーが設定済み

---

## アカウント登録・ログイン

### 新規登録

1. ブラウザで `http://localhost:3000` を開く
2. ランディングページの「無料で始める」ボタンをクリック
3. 認証ページで「新規登録」タブを選択
4. 表示名・メールアドレス・パスワードを入力
5. 登録完了後、自動でダッシュボードに遷移

### ログイン

1. 認証ページで「ログイン」タブを選択
2. 登録済みのメールアドレスとパスワードを入力
3. ログインボタンをクリック

---

## ダッシュボード（`/dashboard`）

ログイン後に表示される概要ページです。

- **プロジェクト数**: 登録済みのプロジェクト数
- **開発記録**: 開発ログの合計件数
- **最近のプロジェクト**: 直近に更新したプロジェクトへのショートカット
- **技能上位**: クイズ回答に基づくスキルスコア（準備中）

---

## プロジェクト作成（`/projects/new`）

ポートフォリオに掲載するプロジェクトを登録します。

入力項目:
- **プロジェクト名**（必須）
- **概要**
- **使用技術**（カンマ区切り）
- **GitHub リポジトリ URL**
- **デモ URL**
- **ステータス**（進行中 / 完了 / 保管）
- **公開設定**（公開ポートフォリオに表示するか）

作成後は自動でプロジェクト詳細画面に遷移します。

---

## 開発ログ（`/projects/:id`）

プロジェクトの詳細と開発ログの一覧を確認できます。

### 手動でログを追加

- 作業種別（コード生成 / デバッグ / 設計判断 / 学び / エラー解決）を選択
- 要約・詳細・使用技術を記録

### MCP サーバーから自動記録（任意）

MCP サーバーを使うと、AI ツールの開発過程を自動で記録できます。
詳細は `MCP_SERVER.md` を参照してください。

---

## プロフィール設定（`/settings`）

公開ポートフォリオに表示される情報を編集できます。

- 表示名
- ユーザー名（公開 URL 用）
- 自己紹介
- GitHub URL

ユーザー名を設定すると、公開 URL は `/p/:username` になります。

---

## 公開ポートフォリオ

1. プロジェクトの「公開設定」をオンにする
2. プロフィールでユーザー名を設定する
3. 以下の URL で公開ページが閲覧可能になります

- 公開ポートフォリオ: `/p/:username`
- 公開プロジェクト詳細: `/p/:username/:projectId`

---

## MCP トークン管理（`/settings`）

MCP サーバーから開発ログを送信するための長寿命トークンを発行できます。

1. 設定ページの「MCP トークン」で新しいトークンを発行
2. 表示されたトークンをコピー（再表示できません）
3. `~/.mex/config.json` を作成してトークンを設定

```json
{
  "api_url": "http://localhost:8000/api",
  "api_key": "<your_api_key>",
  "ai_tool": "claude_code"
}
```

不要になったトークンは「無効化」ボタンで失効できます。

## 画面遷移フロー

```
ランディング (/)
  │
  ├─ [無料で始める] → 認証 (/auth)
  │                      ├─ 登録 → ダッシュボード (/dashboard)
  │                      └─ ログイン → ダッシュボード (/dashboard)
  │
  └─ ※認証済みの場合 → ダッシュボード (/dashboard)
                          │
                          ├─ [新規プロジェクト] → プロジェクト作成 (/projects/new)
                          │                       └─ 作成後 → プロジェクト詳細 (/projects/:id)
                          │
                          ├─ [プロジェクト詳細] → 開発ログ追加 (/projects/:id)
                          │
                          └─ [設定] → プロフィール / MCP トークン管理 (/settings)
```

## よくある質問

### Q: 公開ポートフォリオは誰でも閲覧できますか？

はい。公開設定にしたプロジェクトと、ユーザー名を設定したプロフィールが対象です。

### Q: トークンを失効したらどうなりますか？

MCP サーバーからの投稿は拒否されます。必要に応じて新しいトークンを発行してください。
