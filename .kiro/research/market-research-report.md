## 2. 競合サービスの料金体系

### A. エンジニアスキル可視化・採用プラットフォーム

| サービス | 個人向け料金 | 企業向け料金 | 収益モデル | ユーザー数 |
|----------|-------------|-------------|-----------|-----------|
| **LAPRAS** | 無料 | Self Plan: 月額制 / Premium Plan: 月額+成功報酬 / JOB BOARD: 年収20%成功報酬 | B2B（企業課金） | エンジニア登録多数 |
| **Findy** | 無料 | 月額4.5〜10万円 + 成功報酬30-35% | B2B（企業課金） | - |
| **Forkwell** | 無料 | Standard: 6ヶ月60万〜 + 成功報酬20-25% / 0%Plan: 6ヶ月190万〜 | B2B（企業課金） | - |

**重要な洞察**: LAPRAS、Findy、Forkwellはすべて**個人ユーザーは無料**で、**企業（採用側）からの課金**で収益を得るB2Bモデル。MEXのB2Cサブスクモデルとは根本的に異なる。

#### LAPRASの詳細（最も近い競合）

- GitHub、Qiita、note、Twitter等から自動でアウトプット情報を収集
- 技術力スコアを算出（3.0が平均、3.5以上が上位4%）
- ポートフォリオ自動生成（個人は完全無料）
- 連携サービス数: 国内最多の10サービス

**MEXとの差別化ポイント**: LAPRASはWeb上の既存アウトプットを集約するのに対し、MEXは開発プロセス自体をMCP Server経由でリアルタイム記録する。またクイズによるスキル証明はLAPRASにない独自機能。

### B. プログラミング学習プラットフォーム（B2C比較対象）

| サービス | 無料プラン | 有料プラン | ターゲット |
|----------|-----------|-----------|-----------|
| **Progate** | 初級レッスン無料 | 月額1,490円（年額プラン: 月990円） | 初心者〜初級者 |
| **paiza ラーニング** | 一部無料 | 月額1,078円 | 初心者〜中級者 |
| **AtCoder** | コンテスト無料 | アルゴリズム実技検定（PAST）: 有料受験 | 中級者〜上級者 |
| **Recursion** | 一部無料 | 月額約5,000〜9,000円 | CS基礎を学びたい人 |

**MEX（¥780/月）の価格ポジション**: Progateの年額プラン（¥990/月）より安く、paiza（¥1,078/月）よりも安い。学習系サービスの中では手頃な価格帯。

## 3. ポートフォリオサービスの需要

### エンジニア採用市場の背景

| 指標 | 数値 |
|------|------|
| IT人材不足予測（2030年） | 最大79万人（経済産業省） |
| IT技術関連職の有効求人倍率 | 3.17倍 |
| 2025年上半期エンジニア転職市場 | 売り手市場継続（求人数・転職者数ともに前年超） |
| 新卒エンジニア初任給（2025年卒） | 400〜450万円が最多（26.1%） |
| ITエンジニア平均年収 | 約462万円（doda 2024年12月時点） |
| 初任給引き上げ企業 | 7割超 |

### ポートフォリオの位置づけ

- **経験者（中途）**: ポートフォリオの提出が求められるケースは少ない。GitHub、Qiita等を職務経歴書に記載する程度
- **未経験・新卒**: ポートフォリオは差別化の手段として重要。実務経験がない分、学習成果を補完する役割
- **2025年トレンド**: 採用側でコーディングテスト、ポートフォリオ、GitHub提出など「スキルの可視化」手段の採用が増加

**MEXへの示唆**: 経験者マーケットではポートフォリオ需要は限定的だが、**新卒・未経験→ジュニア転職層**ではポートフォリオが差別化の武器となる。MEXのターゲット（CS学生、ブートキャンプ卒業生、ジュニア開発者）はまさにこの層。

## 4. AI × スキル証明市場

### 市場動向

- **AI資格需要増加**: Google Cloud AI認定、Microsoft AI-900、AWS Machine Learning等の資格取得が転職市場での競争力に直結
- **スキル証明の3階層化**: 企業がAI人材を「一般ユーザー層」「パワーユーザー層」「開発・運用専門層」に分類して育成
- **エンジニアスキル要件の変化**: 2026年に向けて「AI、ローコードツール、ライブラリを統合する力」が求められる

### MEXの差別化要素

| 既存のスキル証明 | MEXの独自アプローチ |
|-----------------|-------------------|
| 静的な資格証明（試験一回のスナップショット） | 継続的な開発プロセスの記録 |
| Web上の既存アウトプット集約（LAPRAS） | MCP Server経由のリアルタイム開発記録 |
| 自己申告ベースのスキルシート | クイズによる客観的な理解度証明 |
| コーディングテスト（AtCoder等） | 実際の開発プロセス全体の可視化 |

**ポジショニング**: MEXは「プロセス証明」という新しいカテゴリを創出できる可能性がある。既存サービスが「結果（成果物やスコア）」を証明するのに対し、MEXは「過程（どのように考え、開発したか）」を証明する。

## 5. 支払い意欲（Willingness to Pay）

### 日本のジュニア開発者の課金行動

| カテゴリ | 価格帯 | 代表的サービス |
|----------|--------|---------------|
| プログラミング学習 | ¥990〜¥1,490/月 | Progate, paiza |
| CS基礎学習 | ¥5,000〜¥9,000/月 | Recursion |
| AIコーディングツール | $10〜$19/月（¥1,500〜¥2,900） | GitHub Copilot |
| プログラミングスクール | ¥50,000〜¥60,000/月 | 各種スクール |
| 技術書・オンライン講座 | ¥200〜¥5,000（単発） | Zenn Books, Udemy |

### MEX ¥780/月の価格分析

- **Progate年額プラン（¥990/月）より安い**: 学習ツールの最安価格帯
- **新卒エンジニア月収約25万円の0.3%**: 心理的に負担感が非常に小さい
- **CS学生の場合**: アルバイト時給1,200円×0.65時間分。学生にとってもギリギリ許容範囲
- **日本の課金カルチャー**: サブスク月額¥500〜¥1,500が「気軽に試せる」価格帯（Netflix ¥790、Spotify ¥980との比較）

**支払い意欲の壁**:
1. **無料競合の存在**: LAPRAS（無料）、GitHub（無料）で基本的なポートフォリオは構築可能
2. **ROIの明確さ**: 「このサービスに課金したことで就職/転職が有利になった」という実績が重要
3. **口コミ依存**: 日本市場ではTwitter/X、Zenn、Qiitaでの口コミが課金判断に大きく影響

## 6. 総合分析とMEXへの示唆

### 市場機会

1. **IT人材不足（2030年に79万人）**: エンジニアのスキル証明・採用マッチング市場は拡大が確実
2. **AI時代のスキル証明需要**: 「AIを使える」だけでなく「AIと協働してどう開発するか」のプロセス証明は新しい需要
3. **新卒・ジュニア層の増加**: ブートキャンプ、プログラミングスクール卒業生が増加し、差別化手段としてのポートフォリオ需要が拡大
4. **低価格B2Cモデルの空白**: 既存のスキル可視化サービス（LAPRAS等）はB2B課金モデル。B2Cでの有料スキル証明プラットフォームはほぼ空白

### 競争上の課題

1. **LAPRAS無料モデルとの競争**: 基本的なスキル可視化が無料で利用可能な市場で、MEXが有料であることの正当化が必要
2. **価値の差別化**: 「開発プロセスの記録」と「クイズによるスキル証明」のユニークな価値を明確に伝える必要
3. **ネットワーク効果の構築**: ポートフォリオは「見る人がいて初めて価値がある」ため、企業採用担当者の認知獲得が重要
4. **個人開発SaaSの持続性**: サブスクモデルは継続的な機能アップデートが不可欠

### 月5万円収益達成の実現可能性

| シナリオ | 必要有料会員数 | 達成難易度 |
|----------|---------------|-----------|
| Pro ¥780/月のみ | 約65人 | 中（ニッチ特化で現実的） |
| Pro ¥780/月 + 追加収益源 | 約40人 + α | 中低 |
| フリーミアム転換率5%仮定 | 無料1,300人→有料65人 | 無料ユーザー獲得が鍵 |

**Inkdropの参考データ**: 月$10K+（約150万円）を達成するまでに約3-5年かけ、YouTubeチャンネル（20万登録者）がユーザー獲得の70-80%を担っている。コンテンツマーケティングの重要性を示す。

### 推奨アクション

1. **「プロセス証明」のポジショニング確立**: 結果ではなく過程を証明するという独自価値を明確化
2. **CS学生コミュニティへの浸透**: 大学のCS学部、ブートキャンプとの連携でユーザー基盤構築
3. **成功事例の早期確保**: 「MEXポートフォリオで内定を獲得した」事例が最大の営業ツール
4. **Zenn/Qiitaでのコンテンツマーケティング**: MEXの開発過程自体をコンテンツとして発信し、認知獲得
5. **将来のB2Bピボット検討**: 十分なユーザー基盤構築後、企業向けタレントプール提供への拡張
